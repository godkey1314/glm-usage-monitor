# 01 - 产品介绍

## 产品名称

GLM 用量监控 (GLM Usage Monitor)

## 解决的问题

用户使用 Claude Code 接入智谱 GLM Coding Plan 时，需要频繁查看套餐配额与使用统计。
官方 `glm-plan-usage` 插件通过 Claude Code 子代理查询，但子代理调用 GLM 模型时常因兼容性问题卡在 `Retrying`，体验差。
命令行脚本 `glm-usage` 虽然可靠但输出是纯文本，不够直观。

## 目标

提供一个**独立、美观、实时**的桌面小工具（便携 exe，双击即用），
让用户随时直观掌握 GLM Coding Plan 的用量与配额，无需输入命令。

## 核心功能

1. 总览：总 Token、调用次数、套餐等级
2. 24 小时趋势折线图（Token + 调用次数）
3. 逐小时时段柱状图
4. 配额进度环（5 小时窗口 / 周窗口 / MCP 月度）
5. 自动 60s 刷新 + 手动刷新
6. 自动读取 `~/.claude/settings.json` 中的 token，零配置

## 目标用户

使用 Claude Code + GLM Coding Plan 的开发者（本机单用户场景）。

## 非目标

- 不做多账号管理（单 token 即可）
- 不做历史数据持久化（仅展示官方 API 返回的近 25 小时窗口）
- 不做跨平台（仅 Windows 便携 exe）

# 2026-06-14 GLM 用量监控 - 初始开发

## 背景

用户使用 Claude Code 接入智谱 GLM Coding Plan，需要直观查看用量与配额。
官方 `glm-plan-usage` 插件通过 Claude Code 子代理查询，但子代理调用 GLM 模型时卡在 `Retrying`，体验差。
命令行脚本输出不够直观。用户需要一个**独立、美观、实时**的桌面小工具，并希望**开源**让所有 GLM 用户都能用。

## 目标

- Electron + ECharts 桌面小工具，打包便携 exe
- 实时刷新、图表可视化、精致界面
- **开源版**：用户自填 Token，不依赖 Claude Code / settings.json

## 关键决策

1. **技术栈**：Electron 33 + ECharts 5 + electron-builder（portable）。原因：node 已装、UI 最美、打包稳。备选 Python+PySide6 被用户否决。
2. **接口实测先行**：写 `scripts/probe-api.js` 实测三个 monitor 接口的**原始**字段结构（而非依赖官方脚本处理后的数据）。关键发现：原始 quota 字段是 `type:TOKENS_LIMIT/TIME_LIMIT` + `unit` + `number` + `nextResetTime` + `remaining`，比官方脚本（把两个 token 窗口都叫 "5 Hour"）信息更丰富 → 据此精确区分「5 小时 / 1 周」窗口并展示重置倒计时。
3. **数据层解耦**：`usage.js` 不读配置，`queryUsage(config)` 接收参数，便于测试与开源化。
4. **配置独立**：开源版 `config.js` 读写应用 `userData/config.json`，完全不碰 settings.json。

## 实现内容

- `src/api/usage.js`：monitor 接口调用 + normalize（精确窗口区分）
- `src/api/config.js`：userData 配置读写 + 端点预设
- `src/main.js`：窗口、托盘、IPC（usage:query / config:get / config:save）
- `src/preload.js`：contextBridge 安全暴露
- `src/renderer/`：毛玻璃 UI、ECharts 趋势/柱状/配额环、设置弹窗、自动刷新
- `scripts/copy-echarts.js`：postinstall 拷贝 echarts 到 renderer/lib（离线打包）

## 验证

- 数据层 node 测试：tokens/调用/配额项全部正确
- 渲染链路：`[renderer] rendered: tokens=... limits=3` 确认 IPC→查询→图表全通
- 开源版流程：未配置 → `configured:false` 弹设置；已配置 → 正常查询渲染

# 03 - 系统架构

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 主进程 | Electron (Node.js) | 窗口、托盘、文件/网络访问、IPC |
| 预加载 | preload.js + contextBridge | 安全暴露 API 给渲染层（contextIsolation） |
| 渲染层 | 原生 HTML/CSS/JS | 界面，无框架 |
| 图表 | ECharts 5 | 趋势折线、柱状图、配额环 |
| 打包 | electron-builder | 便携 exe (portable target) |

## 分层与职责

```
┌─────────────────────────────────────────────┐
│  渲染层 (renderer/)  —— 纯展示 + 交互        │
│  index.html / styles.css / renderer.js       │
│  · 数据卡片、ECharts 图表、设置弹窗、刷新     │
│  · 通过 window.api 调用主进程（不直接联网）   │
└──────────────────┬──────────────────────────┘
                   │ IPC (invoke)
┌──────────────────▼──────────────────────────┐
│  主进程 (main.js)  —— 调度                   │
│  · 创建窗口、系统托盘                         │
│  · ipcMain: usage:query / config:get / save  │
└──────────────────┬──────────────────────────┘
                   │ require
┌──────────────────▼──────────────────────────┐
│  数据层 (api/)                               │
│  · config.js: 读写 userData/config.json      │
│    （用户自填 token，不读任何本地 json）       │
│  · usage.js: queryUsage(config) 调 monitor API│
└─────────────────────────────────────────────┘
```

## 配置来源（开源版核心设计）

**用户在本应用设置界面填写 Token + 选择端点**，保存到应用自己的配置文件：

- 路径：`app.getPath('userData')/config.json`
  - 开发模式：`%AppData%\glm-usage-monitor\config.json`
  - 打包后：`%AppData%\GLM用量监控\config.json`
- 内容：`{ "token": "...", "endpoint": "zhipu" | "zai" }`
- `baseUrl` 由 `endpoint` 派生（config.js 的 ENDPOINTS 预设）

**不读取** `~/.claude/settings.json` 或任何环境变量——彻底独立，任何用户填自己的 Token 即可用。

## 数据流

1. 渲染层启动 → 调 `window.api.queryUsage()`
2. 主进程 `usage:query` → `readConfig()` 取配置
3. 若无 token → 返回 `{ ok:false, needConfig:true }` → 渲染层弹出设置
4. 有 token → `queryUsage(config)` 并发请求三个 monitor 接口，规整返回
5. 渲染层更新卡片 + 重绘 ECharts
6. 每 60s 自动重复（setInterval）

## 安全边界

- `contextIsolation: true`，`nodeIntegration: false`
- 渲染层**不能**直接访问文件/网络，只能通过 preload 白名单方法
- Token 仅在主进程内存流转，不落日志、不传渲染层原文

## 打包

electron-builder → portable target → 单个 `GLM用量监控 1.0.0.exe`，双击自解压运行，无需安装。
